# Стратегия миграции: От Legacy MVP к Enterprise-платформе

## 1. Предпосылки (Why Symfony?)
Текущая версия платформы (Port 8080), написанная на процедурном PHP, успешно выполнила роль MVP:
*   Подтверждена бизнес-модель «Цифровых паспортов».
*   Налажена интеграция с хранилищем CephFS.
*   Реализованы продажи оборудования на сумму более 2 млн руб.

**Цель миграции на Symfony 8.3:**
- **Security:** Внедрение 2FA и строгой системы прав доступа (RBAC).
- **Scalability:** Использование Domain-Driven Design (DDD) для расширения функционала.
- **Maintainability:** Стандартизация кода для командной разработки.

## 2. Стратегия "Strangler Fig" (Душитель)
Миграция проводится без остановки текущих бизнес-процессов по следующему плану:

### Этап 1: Инфраструктурный мост (Текущий статус)
- [x] Развертывание параллельного стека Symfony на порту 8000.
- [ ] Настройка общего доступа к базе данных PostgreSQL 15.
- [ ] Конфигурация Nginx как Reverse Proxy для бесшовного переключения между версиями.

### Этап 2: Вынос "Цифрового паспорта"
- [ ] Перенос логики генерации экспертных отчетов в Symfony.
- [ ] Интеграция с Ollama (AI-Ops) для автоматизированного анализа SMART и стресс-тестов внутри нового фреймворка.
- [ ] Реализация публичных карточек товаров с использованием шаблонизатора Twig.

### Этап 3: Перенос Админ-панели и Безопасности
- [ ] Внедрение аутентификации через Symfony Security Bundle.
- [ ] Реализация Audit Trail (логирование всех действий с ценами и остатками).
- [ ] Полное отключение Legacy-модулей на порту 8080.

## 3. Риски и их минимизация
*   **Консистентность данных:** Обеспечивается за счет использования единой БД PostgreSQL и механизма миграций Doctrine.
*   **Доступность:** Старая версия остается ведущей (Master) до полной стабилизации нового ядра.
*   **Безопасность:** Доступ к Legacy-панели ограничивается на уровне IP во время переходного периода.
