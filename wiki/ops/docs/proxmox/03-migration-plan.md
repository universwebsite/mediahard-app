# Стратегия миграции: От Legacy MVP к Enterprise-платформе Symfony 7

Документация по модернизации ИИ-платформы **MediaHard**, плановому выводу из эксплуатации Legacy PHP-бэкенда и бесшовному переносу данных по стратегии *Strangler Fig* [v0.30.0].

---

## 1. Предпосылки модернизации (Why Symfony?)

Текущая версия платформы (Port `8080`), написанная на процедурном PHP, успешно выполнила роль MVP:
*   Подтверждена бизнес-модель «Цифровых паспортов».
*   Налажена интеграция с распределенным хранилищем CephFS.
*   Реализованы продажи ИТ-оборудования на сумму более 2 млн руб.

**Цели миграции на современный стек Symfony 7 / PHP 8.3:**
*   **Security:** Внедрение встроенной двухфакторной аутентификации (2FA), криптографических ключей и строгой системы прав доступа (RBAC/Symfony Security).
*   **Scalability:** Использование Domain-Driven Design (DDD) для расширения функционала и асинхронной обработки очередей (Symfony Messenger).
*   **Maintainability:** Стандартизация кодовой базы для командной Enterprise-разработки и оркестрации сред.

---

## 2. Стратегия "Strangler Fig" (Вытеснение Legacy)

Миграция проводится без остановки текущих бизнес-процессов (Zero-Downtime Migration) параллельно с обкаткой инфраструктуры Proxmox VE 9.

### Этап 2.1. Инфраструктурный мост (Текущий статус)
- [x] Развертывание параллельного стека Symfony 7 на порту `8000` в локальной ВМ Hyper-V.
- [ ] Настройка общего доступа к базе данных PostgreSQL 16 (контейнер `database` в Docker Compose).
- [ ] Конфигурация Nginx Reverse Proxy для бесшовного проксирования и переключения между версиями.

### Этап 2.2. Вынос модуля "Цифровой паспорт"
- [ ] Перенос монолитной логики генерации экспертных отчетов оборудования в сервисы Symfony.
- [ ] Интеграция с локальным инференсом Ollama AI (Llama-3/Llava) для автоматизированного анализа SMART и стресс-тестов внутри нового фреймворка с использованием векторных инструкций **AVX2**.
- [ ] Реализация быстрых публичных карточек товаров с использованием шаблонизатора Twig.

### Этап 2.3. Перенос Админ-панели и Аудита
- [ ] Внедрение сквозной аутентификации через Symfony Security Bundle.
- [ ] Реализация механизма Audit Trail (строгое логирование всех действий администраторов с ценами, логами и остатками).
- [ ] Полное отключение устаревших Legacy-модулей на порту `8080` и отправка старого репозитория в GitHub-архив.

## 3. Риски и их минимизация (Rollback Runbook)

| Риск / Инцидент | Технический триггер | Стратегия предотвращения (Bypass) |
| :--- | :--- | :--- |
| **Консистентность данных** | Ошибка при маппинге старых таблиц. | Использование единой СУБД PostgreSQL и строгого механизма миграций Doctrine ORM. |
| **Доступность (Downtime)** | Сбой нового ядра при переключении Upstream. | Старая версия остается ведущей (Master) до полной стабилизации и прохождения всех CI/CD тестов. |
| **Утечка секретов** | Попадание паролей в публичный репозиторий. | Исключение файлов конфигураций сред (`.env.local`) из индекса Git и вынос шпаргалок в приватный репозиторий `mediahard-ops`. |
