# 🚀 Архитектура локального Staging-контура: Интеграция Symfony 7 и СУБД PostgreSQL

Этот документ содержит архитектурное обоснование структуры локального контура разработки платформы **MediaHard** на виртуальной машине Hyper-V (`vm212`), а также методологию изоляции секретов и траблшутинга распределенных сред.

---

## 🏛️ 1. Концепция гибридного окружения (Compute & Storage Isolation)

Для минимизации накладных расходов на дисковый ввод-вывод (I/O) и ускорения цикла обратной связи (Feedback Loop) на этапе Staging внедрена гибридная топология:
*   **Слой вычислений (Compute Layer):** Ядро фреймворка Symfony 7 работает нативно на хосте Ubuntu 24.04.4 LTS, используя встроенный веб-сервер PHP CLI. Это исключает оверхед файловой системы при монтировании тяжелых Docker Volumes в Windows/Linux.
*   **Слой данных и ИИ (Data & AI Layer):** СУБД PostgreSQL 15 и инференс-движок Ollama изолированы в Docker-контейнерах.

---

## 🗄️ 2. Паттерн «База данных как донор данных» (Database Routing)

При переезде со старой Legacy-архитектуры на Symfony 7 критически важно было сохранить целостность существующих B2B-данных на сумму более 3 млн рублей. 

Параллельный запуск двух инстанций СУБД оптимизирован следующим образом:
1.  Дефолтный пустой контейнер `mediahard-app-database-1` (порт `32768`) принудительно гасится для экономии RAM-ресурсов хоста.
2.  Компонент Doctrine ORM в новом Symfony-приложении перенаправляется напрямую через сетевой мост на стандартный порт `5432` к работающему контейнеру-донору старого сайта (`mediahard-db-1`).

## 🛡️ 3. Пост-мортем: Анализ и ликвидация инфраструктурных инцидентов (HTTP 500)

В ходе развертывания контура автоматизации **Docs-as-Code** было зафиксировано и успешно устранено два критических сбоя ядра:

### Инцидент #01: Сбой инициализации Dotenv компонента
*   **Симптомы:** `PathException: Unable to read the "/.env" environment file`.
*   **Причина:** В рамках реализации политики безопасности **Zero-Leakage** файл `.env` был исключен из индекса отслеживания Git (`git rm --cached`). При синхронизации сред файл физически удалился с Linux-хоста, заблокировав старт ядра Symfony.
*   **Решение:** В корень проекта возвращен публичный файл-шаблон `.env` без чувствительных данных (секретов). 

### Инцидент #02: Исключение маршрутизации EnvNotFoundException
*   **Симптомы:** `Environment variable not found: "DEFAULT_URI"`.
*   **Причина:** Компонент `symfony/routing` требовал жесткой декларации базового URI для сборки абсолютных путей во внешнем контуре MkDocs документации.
*   **Решение:** Все боевые доступы к СУБД и переменные окружения инжектированы через локальный файл **`.env.local`**. Данный файл защищен глобальным конфигурационным файлом `.gitignore` и никогда не попадет в публичный доступ.

Для применения параметров принудительно выполнен сброс кэша компиляции:
```bash
php bin/console cache:clear
```
